# 高専祭デジタルサイネージシステム

高専祭の来場者向けに、展示やイベント情報をリアルタイムで表示するデジタルサイネージシステムです。

## プロジェクト概要

- **目的**: 高専祭の来場者体験向上、情報の効率的な伝達、実践的な開発経験の獲得。
- **ターゲットユーザー**: 主に来場者、緊急時は学生や教員。
- **技術スタック**:
  - **フロントエンド**: Next.js, Tailwind CSS
  - **バックエンド**: Supabase
  - **データベース**: SQLite (ローカル開発用), Prisma
  - **UI**: Shadcn UI
  - **ハードウェア**: Raspberry Pi 4B/5

## 開発環境のセットアップ

1. **プロジェクトのクローン**:

```
git clone [リポジトリ URL]
cd kosen-festival-signage-system
```

2. **依存関係のインストール**:

```
npm install
```

3. **データベースのセットアップ**:

`.env`ファイルを作成し以下のように環境変数を定義する。

```
DATABASE_URL="file:./dev.db"
```

DBのスキーマ変更後や、DBの削除後には以下コマンドを順に実行し、DBの初期化を行う。

```
npx prisma db push
npx prisma generate
npx prisma db seed
```

prismaのGUI管理には

```
npx prisma studio
```

を実行することで行える。

4. **開発サーバーの起動**:

```
npm run dev
```

- 開発サーバーは `http://localhost:3000` で起動します。
- Prisma Studio を使ってデータベースを操作するには、別のターミナルで `npx prisma studio` を実行します。

## 開発フロー

### 1. Issueを建てる

NewIssueより，バグ報告，改善・リファクタ，実装・追加のどれかからイシューを作成します．作成時にはできるだけ他の人でもわかりやすいように画像などを貼り付けます．

### 2. ブランチを切る

Issueで作成した内容を元に，mainブランチからブランチを切ります．ブランチ名は以下の表のとおりです．
| 目的 | ブランチ名 |
| --- | --- |
| 開発 | feature/[機能名] |
| バグ修正 | fix/[バグ内容] |
| リファクタリング | refactor/[内容] |
| ドキュメント | docs/[内容] |

### 3．開発

建てたイシューに対して開発を行い，コミットを行います．コミットメッセージにはGit

- **Git ブランチモデル**: main ブランチは常に本番環境にデプロイ可能な状態を保ちます。新機能の開発は、feature/[機能名] のようなブランチを切って行い、開発が完了したらプルリクエストを作成します。
- **コミットメッセージ**: feat: 新機能の追加 や fix: バグの修正 のような、わかりやすい形式で記述します。

## 主要なコマンド

- **開発サーバー起動**: `npm run dev`
- **ビルド**: `npm run build`
- **データベース更新**: `npx prisma db push`
- **データベース管理**: `npx prisma studio`
